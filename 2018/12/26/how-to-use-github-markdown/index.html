how to use github&markdown

issue:讨论bug,疑问等,标题要突出主题
assigned:由仓库管理员指派,改issue有被指派的人维护,可以关闭改issue
pull request:请求合并自己的更改
pull request流程:首先要create branch,然后在自己创建的branch里更改,
然后点击create pull request, base选择master,compiler选择自己的branch,写commit message,
接着create pull request.等待或讨论这个变更,被通过后就可以 合并merge,合并完后那个分支branch
就没用了,可以删掉delete branch.
